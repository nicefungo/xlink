## 标准操作流程

```
1. cd /home/admin/xlink && make clean && make all 2>&1
2. make test 2>&1
3. git log --oneline -5  (检查自上次以来的变更)
4. 如果 src/ 有变更 → 做代码审查
5. 如果 src/ 无变更 → 跳过代码审查
6. 读 docs/future-plans/index.md → 更新决策日志 + 日期
7. 读 docs/known-issues.md → 检查是否有变化
8. git add -A && git commit -m "docs: Round N — ..."
9. 输出简洁摘要（不超过 500 tokens）
```

## 输出格式

```
## xlink Round N — YYYY-MM-DD

**Build:** make all → N warnings | make test → N/N PASS
**Changes:** [本轮做了什么]
**Status:** [关键指标]
```
